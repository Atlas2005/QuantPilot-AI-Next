# Thirty-Day Capital-Test MVP Plan

R1 redirects the next 30 days toward a v0.1 Capital-Test MVP.

The MVP target is controlled capital-test readiness: real data-backed candidates after approved gates, sandbox-tested feasibility, paper tracking readiness, and auditable order drafts. It is not live trading and not a profitability claim.

## Days 1-5: R1 Architecture Reset

- finalize profit-first integration architecture
- create the integration replacement matrix
- define Market Reality Sandbox target
- define Capital-Aware Fast Compounding Mode
- define multi-agent target roles
- preserve all no-live-trading safety gates

## Days 6-10: Provider And Data Gate Refresh

- select controlled provider probe sequence after review
- keep raw data out of tracked paths
- define timestamp and provider failure audit fields
- review AkShare, Baostock, and Tushare against license and reliability needs

## Days 11-16: Market Reality Sandbox Skeleton

- specify sandbox input and output contracts
- model A-share T+1, lots, price limits, suspension, costs, slippage, and liquidity
- define rejected-order and partial-fill simulation requirements
- benchmark RQAlpha, vectorbt, Backtrader, and Hikyuu for fit without selecting final engines

## Days 17-22: Signal And Validation Gate

- connect approved local fixtures or approved provider samples to signal review
- require OOS and walk-forward design before evidence claims
- add leakage, capacity, turnover, and cost checks
- keep factor analytics packages as benchmark/prototype candidates only

## Days 23-27: Capital-Aware Candidate Packet

- define capital and account permission snapshot schema
- define sandbox order draft schema
- add risk, concentration, and rejection reason summaries
- produce report-ready candidate packet format

## Days 28-30: Review And Capital-Test Readiness Decision

- run full automated validation
- review safety flags and no-execution boundaries
- produce a go/no-go packet for controlled paper tracking or later capital-test preparation
- keep broker connection and live execution out of scope unless a future approved phase changes the boundary
