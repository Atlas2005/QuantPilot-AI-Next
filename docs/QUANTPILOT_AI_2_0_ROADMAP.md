# QuantPilot-AI 2.0 Roadmap

## Phase 0A: Planning Package Only

Create planning documents, policies, handoff files, and review packet. No implementation code.

## Phase 0B: Clean Project Skeleton and Minimal CI

Create a clean repository skeleton and minimal documentation/format validation after review.

## Phase 1: Open-Source Candidate Registry

Create a registry for candidate projects and evaluation status.

## Phase 2: Core Contracts

Define core input/output contracts before implementation.

## Phase 3: Data Contracts and Local Fixtures

Define data contracts and approved local fixtures without market-data fetching.

## Phase 4: Controlled Data-Source Prototypes

Prototype data-source access behind contracts after approval.

## Phase 5: A-Share Market Rule Engine

Model A-share rules including T+1, lot size, limit behavior, suspension, ST treatment, fees, stamp duty, slippage, liquidity, and small-capital constraints.

## Phase 6: Backtest Engine Evaluation

Evaluate mature backtest/research engines through adapters and contract tests.

## Phase 7: Alpha Engine and Factor Validation

Create factor validation workflows with evidence-gated methodology.

## Phase 8: Strategy Tournament

Create controlled strategy comparison and promotion rules.

## Phase 9: Walk-Forward / OOS Validation

Create sample-out, walk-forward, and out-of-sample validation workflows.

## Phase 10: Paper Trading Feedback Loop

Create paper feedback workflows after research and validation foundations are stable.

## Phase 11: Agent Orchestration

Introduce agent orchestration only after data, contracts, validation, and backtest foundations are stable.

## Phase 12: Dashboard / Productization / Commercial Readiness Review

Prepare dashboard, productization, licensing review, operational review, and formal commercial/trading readiness review.
