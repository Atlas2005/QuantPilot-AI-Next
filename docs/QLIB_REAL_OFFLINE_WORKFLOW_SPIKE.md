# P41 Qlib Real Offline Workflow Spike

P41 moves Qlib from metadata-only handoff toward a real offline workflow boundary.

This is still standard-library-only and deterministic by default. It does not install Qlib, require Qlib for pytest, run qrun by default, fetch data, call brokers, call LLM runtimes, or claim real profitability.

## Objective

P41 answers:

- did Qlib move beyond metadata-only handoff?
- was a Qlib-style local dataset spec produced?
- was a Qlib-style workflow config produced?
- were factor candidates mapped from existing alpha, AI, and replay concepts?
- was an offline evaluation spike computed?
- was Qlib runtime disabled by default?
- is optional dependency status explicit?
- did mixed stock+ETF remain covered?
- did comparison against the P40 AI-shadow-adjusted replay baseline complete?
- is the safety barrier still at or below `140%`?

## Dataset Bridge

The dataset bridge accepts approved local provider-export style records or deterministic fixtures and produces a Qlib-style local dataset spec.

It validates:

- explicit source type
- explicit instrument kind
- mixed stock+ETF coverage when mixed mode is enabled
- ETF category for ETF rows
- unique symbol/date rows
- OHLCV-like fields
- no future rows beyond the evaluation window
- close and volume presence
- deterministic date normalization

The spec preserves provider name, source type, stock/ETF counts, field mapping, calendar assumptions, cost model assumptions, benchmark candidate, and known limitations.

## Workflow Config

The workflow config includes:

- dataset id
- universe name
- benchmark
- label expression placeholder
- factor feature list
- train/validation/test window placeholders
- cost model assumptions
- execution assumptions
- runtime status
- `qrun_disabled_by_default = true`

Qlib runtime execution remains disabled by default.

## Factor Mapping

P41 maps existing project concepts into Qlib-style factor candidates:

- `momentum_proxy`
- `volatility_proxy`
- `liquidity_proxy`
- `cost_drag_proxy`
- `etf_category_proxy`
- `capital_fit_proxy`
- `ai_shadow_preference_proxy`

Each factor includes required fields, instrument scope, leakage-control notes, expected direction, and known limitations.

## Offline Evaluation

P41 computes deterministic proxy scores:

- instrument count
- stock count
- ETF count
- factor count
- candidate score
- cost-adjusted score
- tradability score
- small-capital fit score
- Qlib runtime status
- disabled-by-default runtime flag
- warnings

This is not a real Qlib backtest and does not claim real profitability.

## P40 Comparison

P41 compares the Qlib-style offline evaluation against the P40 AI-shadow-adjusted mixed ETF replay baseline and reports:

- optional runtime readiness later
- mixed stock+ETF support
- factor alignment with AI shadow recommendations
- cost-aware score agreement with P40 replay direction
- advisory promotion warnings
- whether Qlib should move to the next optional runtime spike

## Safety Boundary

P41 does not:

- connect to brokers
- read real accounts
- place real orders
- import broker SDKs
- call DeepSeek
- call OpenAI
- make network calls in default tests
- fetch live or remote provider data in default tests
- install packages
- modify project dependencies
- run Qlib or qrun by default
- require Qlib for default tests
- add generic preflight-only gates
- claim real profitability

## Recommended Next Step

Install an isolated optional Qlib environment outside default pytest, then run a controlled manual Qlib runtime spike against the local dataset/workflow metadata generated by P41.
