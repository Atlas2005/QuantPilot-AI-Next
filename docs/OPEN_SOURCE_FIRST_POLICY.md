# Open-Source-First Policy

Before building any major module from scratch, QuantPilot-AI 2.0 must review mature open-source alternatives.

## Required Evaluation Dimensions

- license and commercial risk
- maintenance status
- Windows compatibility
- A-share fit
- integration cost
- reliability
- testability
- adapter compatibility
- contract-test compatibility
- operational risk

## Decision Options

Each candidate may later be classified as:

- `adopt_directly`
- `wrap_with_adapter`
- `borrow_architecture_only`
- `prototype_required`
- `defer_until_foundation_ready`
- `avoid_for_now`

Step 0A records the universe and evaluation rules only. It does not make final technical selections.
