# R1 Module Kickoff Review: Profit-First Integration Architecture Reset

## Objective

Reset QuantPilot-AI-Next toward a profit-first, integration-first, A-share-focused, multi-agent architecture for a 30-day v0.1 Capital-Test MVP.

R1 is architecture reset only. It does not approve live trading, broker connectivity, order execution, package installation, or real data fetching.

## Why This Module Exists

The prior phase trail built valuable safety gates, contracts, candidate registries, market-rule foundations, and provider readiness checks. R1 reframes the next work around capital-test readiness instead of slow policy-only expansion.

Future modules should be judged by whether they advance:

- real data-backed signal validation after gates
- Market Reality Sandbox feasibility
- capital-aware trade candidate review
- sandbox order drafts
- paper tracking readiness
- execution-readiness review

## Scope

- add R1 architecture docs
- create integration reset matrix
- add standard-library loader and validator
- add tests for R1 safety invariants
- update project state, decisions, handoff, review packet, and README index

## Out Of Scope

- no dependency installation
- no real data fetch
- no provider adapter
- no broker connection
- no live trading
- no order execution
- no profitability claim
- no final technical selection
